We are proud to announce the latest version of Unikraft, `v0.19.1`!
This release comes with bug fixes and improvements.

⚠️  Be aware of the following potentially breaking changes:

In `libukbitops`:

* `uk_ffs()` / `uk_ffsl()` have been updated to return values consistent with libc's `ffs()` / `ffsl()` and `__builtin_ffs()` / `__builtin_ffsl()`.
* Previous exact behavior of `uk_ffs()` / `uk_ffsl()` is now provided by `uk_lssb()` / `uk_lssbl()`.
* `uk_fls()` / `uk_flsl()` are replaced by `uk_mssb()` / `uk_mssbl()`.

Here is the full list of updates:

* `lib/vfscore`: Harmonize handling of paths in \*at syscalls ([#1647](https://github.com/unikraft/unikraft/pull/1647)) by @andreittr
* `lib/posix-process`: Fix clone() return value on thread create ([#1649](https://github.com/unikraft/unikraft/pull/1649)) by @michpappas
* `lib/uksched`: Rework waitqueues to use thread data ([#1651](https://github.com/unikraft/unikraft/pull/1651)) by @andreittr
* `lib/ukbitops`: Rework bit find ops for consistency ([#1652](https://github.com/unikraft/unikraft/pull/1652)) by @andreittr
* `lib/ukboot`: Skip graceful shutdown on last child release ([#1653](https://github.com/unikraft/unikraft/pull/1653)) by @michpappas
* `lib/ukrandom`: Fix undefined behavior ([#1655](https://github.com/unikraft/unikraft/pull/1655)) by @michpappas
* `lib/posix-process`: Fix stack corruption in signal delivery ([#1657](https://github.com/unikraft/unikraft/pull/1657)) by @michpappas
* `lib/vfscore`: Fix uninitialized mutex in dentry ([#1675](https://github.com/unikraft/unikraft/pull/1675)) by @andreittr
